- Unix users, password and group files #Light #Unix
	- `/etc/passwd`
		- Rows `username,x,userid,groupid,name,homedir,shell`
		- The x tells the password is stored in shadow file
		- `/sbin/nologin` denies login, mainly for service related users
	- `/etc/shadow`
		- Stores the password hash of each user and contains !! for blank passwords
	- `/etc/group`
		- Rows `groupid,x,users`
	- `useradd USERNAME`
		- `-s`: Default shell
		- `-d`: Home dir
		- `-m`: Populate home dir
	- `usermod USERNAME`
		- `-c`: Set name
		- `-m`: Move homedir
	- `userdel USERNAME`
		- `-r`: Remove home dir
	- `/etc/login.defs`
		- Contains password policies
			- `PASS_MAX_DAYS`: Password expiry date. Default is 99999
			- `PASS_MIN_DAYS`: Minimum days before password change. Default is 0
			- `PASS_MIN_LEN`: Minimum password size. Default is 5
			- `PASS_WARN_AGE`: Number of days after password change warning sent. Default is 7
	- `/etc/pam.d/system-auth`
		- Pluggable Authentication Module
		- Uses pam_cracklib.so
			- `retry`
			- `minlen`
			- `ucredit`: Uppercase limit. -1
